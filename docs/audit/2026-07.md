# Correctness Audit & Fixes — July 2026

A full-codebase correctness audit of the initial open-source release
(`db65144`), followed by a fix pass. Seven parallel reviewers swept the
subsystems (lexer/parser, sema, codegen/interpreter, runtime core/engine,
ta/math, collections/str, strategy/drawing); every finding below was verified
against the source or reproduced by execution before being fixed. Ground truth
throughout is TradingView Pine Script v6 (reference manual + documented broker
emulator behavior).

**Baseline:** 406 tests passing, 0 failing — yet all of the bugs below existed.
Coverage was concentrated on historical, single-shot runs with clean (na-free)
data, which is why they survived.

**End state:** 530 tests passing, 0 failing, typecheck clean. The count
includes 116 PineTS parity tests that were silently skipping before
(`bun install` had never been run in CI-equivalent form locally, so the
optional PineTS dev-dependency was absent) — the numeric fixes are now
cross-validated against a second Pine implementation on every run.

---

## 1. Critical — wrong results or crashes in both backends

### 1.1 `na` was truthy in every condition context

The runtime `NA` sentinel is a frozen object (`{__na: true}`), and both
backends used raw JS truthiness for `if`, `while`, ternaries, `and`/`or`, and
subject-less `switch`. An na bool therefore *executed* the branch it should
have skipped:

```pine
var bool trigger = na
if trigger        // executed! Pine v6: na coerces to false
    c := 1
```

Reachable from `bool x = na`, if-expressions without `else`, unconfirmed
`request.security` bars, and bool history reads. Comparisons were safe (they
route through na-aware `$.eq`/`$.lt`/…); only bare bool conditions were
affected.

**Fix:** every condition context in `codegen/emit.ts` and
`interp/interpreter.ts` now coerces through `$.toBool` (na → false, matching
v6's "bools cannot hold na" rule). `$.not` in `runtime/context.ts` coerces the
same way, so `not na` is `true`. Laziness of `and`/`or` is preserved
(`$.toBool(l) && $.toBool(r)`).

### 1.2 `break`/`continue` inside a `switch` in a loop → compile failure

Switch *statements* lowered to the same IIFE as switch *expressions*, so a
loop-control statement inside a case body emitted `break;` inside an arrow
function — illegal JS. `new Function` threw a raw `SyntaxError`, and since
`compile()` always runs the emitter, the script failed for **both** backends.
Worse, the interpreter's own path would have silently *swallowed* the `break`
(`switchValue`/`blockValue` discarded the control-flow sentinels), so the two
backends were non-equivalent by construction.

**Fix:** statement-position switches now lower to a plain if/else chain
(`emit.ts` `switchStmt`); the interpreter propagates the `BREAK`/`CONTINUE`
sentinels through `blockValue` and the `Switch` statement case.

### 1.3 The realtime tick path was systematically broken

Historical replay was solid; the developing-bar path had four independent
defects (`engine/driver.ts`, `runtime/context.ts`, `runtime/output.ts`):

| Defect | Consequence |
|---|---|
| `onTick` never called `onStrategyBar()`/`onStrategyBarClose()` | strategies queued orders forever; nothing ever filled live |
| Rollback snapshot omitted all broker state | each tick re-executing `strategy.entry` stacked another pending order (10 ticks → 10 orders) |
| Alerts/logs were append-only and never rolled back | a condition true across N ticks of one bar emitted N duplicate alerts |
| `request.security` caches were computed once over historical `allBars` | every `security` call returned `na` forever once live ticks began |

**Fix:** `onTick` now runs the same strategy hook sequence as historical bars;
`RollbackSnapshot` gained `strategy` (deep broker snapshot via new
`StrategyBroker.snapshot()/restore()`) and `alertCount` (alerts truncate on
rollback); the driver syncs the developing bar into `allBars` and invalidates
the security caches each tick; `rollback()` lazily captures a baseline snapshot
so realtime-only usage (no `runHistorical`) is consistent too.

Verified end-to-end: 3 ticks + close on one bar, then another bar — both
backends produce identical output; one alert per bar (not per tick); the
live-bar entry fills at the next bar's open; `security` returns real values on
realtime bars.

### 1.4 `time_close` returned the bar's *open* time

The getter was byte-identical to `time` (copy-paste slip — the function form
`time_close(tf)` was already correct). `time_close - time` plotted 0; countdown
and session logic fired a full bar early.

**Fix:** the getter returns `time + one timeframe`. The one test that had
encoded the buggy value was corrected.

### 1.5 Function inliner variable capture

`expand()` bound parameters as same-named sequential `VarDecl`s with no
renaming, so a later argument referencing a caller variable named like an
earlier parameter captured the fresh binding:

```pine
f(a, b) => a - b
a = 100.0
r = f(1, a)   // engine: 0 (b bound to the new a=1); Pine: -99
```

The method-receiver "bind `this` last" reorder just traded one capture
direction for the other (`v.add(2)` could bind `this` to the float `2`).

**Fix (`sema/inline.ts`):** each argument/default expression now binds to a
fresh `__arg<i>_<expansionId>` temp first; parameters then bind from the temps.
All caller-scope evaluation happens before any parameter name exists — no
capture in either direction. The receiver-reorder hack was deleted.

## 2. High — silent wrong numbers

### 2.1 Type inference feeding `+` dispatch (`sema/analyze.ts`)

Both backends dispatch `+` on the inferred type (string → concat, else
numeric add). Three table bugs corrupted that:

- Everything in `str.*` except `length` was typed string →
  `str.tonumber("2") + 1` compiled to concatenation (`"21"`). Now: `tonumber` →
  float, `pos`/`length` → int, `contains`/`startswith`/`endswith` → bool.
- String-valued builtin members (`syminfo.ticker/prefix/currency/…`,
  `timeframe.period`) were typed float → `syminfo.prefix + syminfo.ticker`
  did numeric addition → NaN. Now typed string (plus bool/int/color members
  fixed alongside).
- A user `method tostring(...)` (any name colliding with a builtin namespace
  function) **hijacked** `str.tostring(...)` calls — the dot-call sugar rewrite
  only excluded collection/drawing method names. Now the receiver must not be a
  builtin namespace ident. This fix alone made two corpus scripts compile that
  had been failing with "Left side of assignment is not a reference".

Also: UDT constructor arguments were analyzed twice, creating phantom duplicate
input declarations (a user override of `"Length"` silently landed on the dead
key while the live one was `"Length (2)"`) and leaking state slots. Args now
analyze exactly once.

### 2.2 Strategy broker emulator (`runtime/builtins/strategy.ts`)

| Bug | Pine behavior | Was |
|---|---|---|
| `strategy.order` opposite-direction | nets against position (long 10 + short 4 → long 6) | full reverse (→ short 4, phantom exit trade) |
| Limit/stop fills vs gaps | bounded by bar open (buy stop 100, open 105 → fill 105) | always at trigger price (impossible fills, flattered backtests) |
| Same-id resubmit | replaces the resting order | stacked duplicates (`exit()` already did replace-by-id — the asymmetry was the tell) |
| Exit bracket after partial fill | spent | stayed armed and re-fired on later bars |
| `strategy.close(id)` | no-op unless id matches the open entry | closed whatever was open |
| Trade profit | includes entry-side commission (pro-rated on partials) | exit-side only (trade list didn't sum to netprofit) |
| profit == 0 trades | `eventrades` | counted as wins; `eventrades` hardcoded 0 |
| Slippage | market **and** stop fills | market only |
| Intrabar stop-vs-limit order | bar-shape heuristic (extreme nearer open first) | always stop first |

Plus the new `snapshot()/restore()` used by realtime rollback (§1.3).

### 2.3 ta/math numerics (`runtime/builtins/ta.ts`, `math.ts`)

- `math.round` rounded ties toward +∞ (JS `Math.round`): `round(-4.5)` gave
  `-4`; Pine rounds away from zero → `-5`. Fixed (also `round_to_mintick`).
- `math.pow(na, 0)` returned 1 (JS `NaN ** 0`); now na.
- One na source value permanently poisoned `ta.bb`'s basis (`sum += NaN` never
  recovers — every other windowed function guarded; this one didn't).
- `ta.kc` smoothed the range with RMA; Pine uses **EMA** of true range (bands
  were systematically wider/slower). `kcw` fixed transitively.
- `ta.change`/`mom`/`roc` skipped na inputs, so "n bars ago" counted non-na
  *samples* instead of bars. Now bar-indexed with na propagation.
- `ta.cmo`/`ta.mfi` stuffed bar 0's change as `0` instead of na — output
  emitted one bar early with a wrong first value.
- `ta.percentrank(na, …)` returned 0 ("lowest of window") instead of na.
- Bare `ta.tr` ≡ `ta.tr(handle_na=false)` per the v6 manual → **na on bar 0**
  (was unconditionally `high − low`). The `handle_na` argument now actually
  reaches the runtime — `normalizeTaArgs` in `codegen/intrinsics.ts` had been
  stripping it, so `ta.tr(true)` and bare `ta.tr` were indistinguishable.
- Woodie pivot used the previous close; Pine uses the **current period's
  open**: `P = (H_prev + L_prev + 2·O_current) / 4`.
- Shrinking a series-int `length` shed only one buffer element per bar
  (`sma(…, len)` returned `sum(5 values)/2` after len dropped 5 → 2). All ~30
  window sheds converted `if` → `while`.
- The 2-arg `ta.vwap(source, anchor)` overload silently ignored the anchor
  (arity heuristic only recognized the 3-arg banded form).

### 2.4 Backend divergences (codegen vs interpreter)

The two backends are supposed to be mutually-checking oracles; four constructs
broke that (all reproduced by running both):

- `continue` inside indexed `for [i, v] in arr` skipped the emitted `idx++` —
  codegen's index desynced from element position (interpreter was right).
  Increment moved to loop top.
- Compound assignment: the interpreter evaluated the RHS *before* reading the
  current value, so `x += (block that reassigns x)` differed (codegen was
  right). Now reads left-to-right.
- Drawing-constructor argument evaluation order: interpreter evaluated the
  opts bag before coordinate args, opposite of the emitted call. Now
  coords-then-opts in both.
- Member access on an na object: codegen threw a raw `TypeError`, interpreter
  yielded na. Codegen now emits `?.` (na), with the assignment-target path kept
  as a direct write (optional chaining is illegal on an LHS).

## 3. Medium — collections, strings, parser

### 3.1 `runtime/builtins/{array,matrix,str,color}.ts`

Convention note: this engine deliberately soft-fails where Pine hard-errors
(conformance tests assert "na, not a crash"). Fixes preserve that but eliminate
JS *wraparound*, which was silently computing on wrong data:

- `array.sort`/`sort_indices` used a numeric-subtraction comparator → string
  arrays never sorted. Now type-aware (strings lexicographic, na pinned last).
- `array.insert(a, -1, x)` spliced from the end (JS wraparound) → no-op.
  `array.fill(a, v, 0, 10)` **grew** a size-3 array to 10 → clamps.
  `str.substring("hello", -3)` slice-wrapped → clamps.
- `array.sum([])` → na (consistent with `avg`); `array.covariance` filters na
  pairs like `variance` does.
- `matrix.mult(matrix, array)` returned an n×1 Matrix; Pine's overload returns
  an array. `matrix.reshape` with a mismatched element count produced
  `undefined` cells → no-op. `matrix.pow` with negative/fractional exponent
  returned the identity matrix → na. `matrix.add_col` on a 0×0 matrix dropped
  the values (asymmetric with `add_row`) → adopts shape. `matrix.trace` on
  non-square → na (was min-diagonal sum).
- `str.format` `{n}` / `{n,number}` / `{n,number,integer}` now apply
  MessageFormat digit grouping (`1340000` → `"1,340,000"` — the Pine manual's
  own example). Explicit patterns keep pattern semantics.
- `str.tonumber` accepted JS extras (`"0x10"` → 16, `"Infinity"`); now a strict
  decimal/scientific grammar. `str.repeat(s, 2.5)` threw a `RangeError`; now
  truncates.
- `color.new(c)` with transp omitted emitted the malformed string
  `"#FF0000NaN"`; now preserves the input's alpha.

### 3.2 Parser (`parser/parser.ts`)

- A block-form `if`/`switch` in *expression* position re-entered the postfix
  loop, so the **next statement's** leading `[` or `(` was parsed as a postfix
  operator on the if-expression:

  ```pine
  v = if useA
      1
  else
      2
  [macdLine, signalLine] = ta.macd(close, 12, 26, 9)  // "expected ]"
  ```

  Block-form if/switch expressions now take no postfix operators (there is no
  legal Pine program where they do).
- UDT fields with dotted builtin types (`chart.point point`) failed with
  "expected a name" — the field gate didn't accept `.` even though
  `isTypeStart()` and statement-level declarations already handled dotted
  chains. Also covers `chart.point[] pts`.

### 3.3 Runtime odds and ends

- Same-symbol detection for `request.security` used unanchored `endsWith`, so
  a request for `WETHUSDT` on an `ETHUSDT` chart was answered with the chart's
  own bars. Now exact match or `:`-boundary only.
- `barstate.isfirst` was hardcoded false on realtime bars — wrong for a
  history-less first live bar (`var`-init patterns never ran).
- `fillColor` dropped `null`, so a later tick of the same bar could never
  *clear* a fill color set by an earlier tick.

## 4. Deliberately NOT changed (known deviations)

These were flagged in the audit and left alone — either documented design
choices or design-level changes that deserve their own pass:

| Item | Why deferred |
|---|---|
| `array.slice` returns a copy; Pine slices are live **views** into the parent | needs a proxy/view design across the whole collections layer |
| "na pauses the window" in `sma`/`stdev`/`dev`/`variance`/etc. | documented deliberate convention in `ta.ts`; TV's own per-function na handling varies |
| `lookahead_off` maps chart bars to the *previous* HTF bucket | documented deliberate non-repainting idiom (TV shows the developing bucket) |
| `/` is always float division; Pine truncates int/int | needs int-type plumbing through both backends' `binary()` dispatch |
| Session VWAP anchors to UTC calendar day | host has no exchange-session/timezone model yet |
| `for`/`while` as expressions (loop returns last body value) | parser feature work; AST nodes exist but were never parseable |
| `binary_search_leftmost/rightmost` extreme-value indices | TV's exact semantics unverified |
| `weekofyear` naive day-count/7 | needs a real ISO/TV week-numbering rule |
| `percent_of_equity` sizing marks open profit at the fill bar's close | look-ahead question needs a design decision on mark-to-market timing |
| Dead `InputNs` in `runtime/builtins/input.ts` | harmless dead code; live implementation is in `context.ts` |

## 5. Verification

- **Full suite:** 530 pass / 0 fail / 1 skip (was 406 pass with 117 skips —
  the PineTS parity suite now actually runs). `tsc --noEmit` clean.
- **Both-backend smoke:** na-conditions, `and`/`or`/`not` with na,
  break-in-switch-in-loop, indexed `continue` — identical, Pine-correct output
  from codegen and interpreter.
- **Realtime replay probe:** multi-tick bars with a strategy + alerts +
  `request.security` — one alert per bar, single position after 6 same-bar
  ticks, live-bar orders filling at next open, security non-na on live bars;
  byte-identical across backends.
- **One parity reclassification:** bare `ta.tr` moved from the "exact match vs
  PineTS" group to the "verified piner-correct (PineTS wrong)" group — PineTS
  returns `high − low` on bar 0 where the v6 manual specifies na. A ground-truth
  test documents the difference.
- **Tests updated (2), both because they encoded a bug:** `time_close === time`
  in `test/coverage.test.ts`; `ta.tr` bar-0 value in `test/ta.test.ts`. New
  regression tests were added for the parser postfix leak, dotted UDT field
  types, inliner capture, method-sugar hijack, str type table, and the tr
  semantics.

## 6. Test-coverage lesson

Everything above coexisted with a fully green suite. The gaps were structural,
and worth keeping closed:

1. **na-injection** — feeds with na gaps and na-typed bools flow through
   conditions, windows, and offsets.
2. **Realtime tick replay** — multi-tick bars exercising rollback of *all*
   mutable state (broker, alerts, drawings), not just series and ta state.
3. **Backend cross-checking with side-effectful expressions** — argument
   evaluation order, compound assignment, loop control through switch.
4. **Parity against a second implementation** — the PineTS suite only ran when
   the optional dependency was installed; it silently skipped otherwise.
   Consider failing CI when the parity suite is skipped.
